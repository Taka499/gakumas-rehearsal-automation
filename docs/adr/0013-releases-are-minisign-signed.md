# Releases are minisign-signed; the embedded public key is the updater's root of trust

The auto-updater will not install a download unless its `.minisig` signature verifies against a minisign (Ed25519) public key compiled into the binary (`src/update/endpoints.rs::PUBLIC_KEY`). The matching secret key lives ONLY on the developer's machine (`~/.minisign/gakumas.key`, password-protected, offline-backed) — never in git, the dist repo, or Cloudflare. This makes the signing key the single root of trust: because the manifest and the release zip share one origin (the `tia-tools/releases` repo behind the tia.run Worker), the pre-existing SHA-256 check cannot stop an attacker who compromises the dist bot PAT or the Cloudflare account from serving malware with a matching hash — but such an attacker still cannot forge a signature. The app installs and restarts with administrator elevation (no fresh UAC prompt), so this closes an admin-RCE supply-chain path (2026-07-08 security review finding #1). Signature verification is MANDATORY (a missing/invalid signature aborts the install) — an optional check would be defeated by simply omitting the signature. Rejected: Windows Authenticode code-signing — it costs a yearly CA certificate, and the cert binds to a legal identity printed in the exe, conflicting with the identity-separation posture of `docs/adr/0011`; its only real gain (SmartScreen reputation) doesn't help an already-unsigned admin-elevated portable app. Authenticode may be layered on later without removing minisign. Consequence: changing `PUBLIC_KEY` is a breaking trust event (a binary can only verify signatures from the key it carries); losing the secret key forces baking a new public key into a future build.

Source: 2026-07-08 distribution security review (subagent) + user decision confirming minisign over Authenticode, /grill-me follow-up session 2026-07-08; implementation plan `docs/EXECPLAN_RELEASE_SIGNING.md` (M1–M3 code-complete, 17 updater unit tests passing; end-to-end tamper-rejection acceptance pending in M4).
